# Туристический телеграм бот 

* Телеграм бот выдает пользователю справочную информацию о введенном городе. Например, пользователь вводит: «Москва», чат-бот отвечает: «Не забудьте посетить Красную Площадь. Ну а в ЦУМ можно и не заходить)))».
* Данные о городах хранятся в базе данных.
* Управление данными о городах (добавление новых городов и информации о них, изменение и удаление любой информации) производится посредством REST WebService. 
* Используемые технологии: SpringBoot, SpringMVC, SpringData, Hibernate, Heroku, PostgreSQL. Для сборки проекта использовался Maven.

## Запуск бота

Для того, чтобы стартовал бот, необходимо запустить класс App, находящийся в src\main\java\com\smuniov\telegrambot\. Необходимые параметры(бота, БД) заданы в файле src\main\resources\application.properties

ник бота: 

@ololotgbot

доступные команды:

/start, /help, /citylist

### Управление данными

Управление данными о городах происходит посредством HTTP запросов на localhost через порт 8080.

Список всей информации:

GET запрос на http://localhost:8080/

Информация о городе по id:

GET запрос на http://localhost:8080/{id}

Создание нового блока данных:

POST запрос на http://localhost:8080/

Тело запроса:

JSON с параметрами name и description
```
{
        "name": "Глубокое",
        "description": "Обязательно попробуйте сгущенку, Глубокская - самая вкусная!"
    }
```

Изменение блока данных:

PUT запрос на http://localhost:8080/{id}

Тело запроса:

JSON с параметрами name и description.

```
{
        "name": "Глубокое",
        "description": "Обязательно попробуйте сгущенку, Глубокская - самая вкусная, а рогачевская вкуснее!"
    }
```

Удаление информации о городе по id:

DELETE запрос на http://localhost:8080/{id}
